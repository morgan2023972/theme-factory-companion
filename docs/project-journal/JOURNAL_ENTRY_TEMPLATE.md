# Modèle d'entrée de journal

- **Date et heure** :
- **Phase** :
- **Tâche** :
- **Type d'événement** : avancement / décision / problème / changement de statut / autre

## Résumé

Résumer brièvement l'événement consigné.

## Changements effectués

Décrire les changements concrets réalisés (documentation, code, configuration, etc.).

## Validations exécutées

Lister les validations réalisées (typecheck, tests, build, validation manuelle, etc. — voir [docs/VALIDATION_STRATEGY.md](../VALIDATION_STRATEGY.md)).

## Résultat

Décrire le résultat obtenu : succès, échec partiel, blocage.

## Problème rencontré

Décrire tout problème rencontré, le cas échéant. Renvoyer vers une entrée dédiée du [registre des erreurs IA](../ai-errors/) si l'erreur provient d'un agent IA.

## Décision prise

Décrire toute décision prise à cette occasion. Renvoyer vers une entrée dédiée du [registre des décisions](../decisions/) si la décision est structurante.

## Prochaine action

Décrire la prochaine action prévue.

## Références Git éventuelles

Indiquer les commits, branches ou pull requests associés à cette entrée, le cas échéant.
