# Journal du projet

Ce journal conserve un historique chronologique des événements significatifs du projet Theme Factory Companion : avancement, décisions, problèmes rencontrés, changements de statut.

## Fonctionnement

- Chaque entrée fait l'objet d'un fichier Markdown distinct dans ce dossier, nommé selon le format `AAAA-MM-JJ-titre-court.md` (ex. `2026-07-12-cadrage-initial.md`). Plusieurs entrées peuvent partager la même date si plusieurs événements notables surviennent le même jour.
- Chaque fichier suit le [modèle d'entrée de journal](JOURNAL_ENTRY_TEMPLATE.md).
- Les entrées sont conservées dans l'ordre chronologique et ne sont pas réécrites après coup : une correction ou un complément fait l'objet d'une nouvelle entrée qui référence la précédente si nécessaire.

## Quand créer une entrée

Une entrée doit être créée à chaque étape significative du projet : fin d'une tâche, changement de statut d'une phase, décision actée, problème rencontré, ou tout autre événement utile à comprendre l'historique du projet a posteriori.

## Différence avec les autres registres

- Les décisions structurantes sont détaillées dans le [registre des décisions](../decisions/), le journal peut simplement y faire référence.
- Les erreurs IA significatives sont détaillées dans le [registre des erreurs IA](../ai-errors/), le journal peut simplement y faire référence.
- Le journal, lui, retrace le déroulé chronologique global du projet, en reliant ces différents éléments entre eux.
